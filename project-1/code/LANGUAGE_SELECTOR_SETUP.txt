## How It Works

1. **Language Selector Position:**
   - Fixed in the top-right corner of every page
   - Always visible and accessible
   - z-index: 1000 (appears above all content)

2. **Language Options:**
   - 🇩🇪 Deutsch (German) - Default
   - 🇬🇧 English
   - 🇪🇸 Español (Spanish)
   - 🇫🇷 Français (French)
   - 🇵🇹 Português (Portuguese)

3. **How to Use:**
   - Open any page on your website
   - Look for the language selector in the top-right corner
   - Click the dropdown and select your preferred language
   - The page content updates instantly
   - Your choice is saved and remembered on all pages

4. **Programmatic Control:**
   You can also change language from JavaScript:
   ```javascript
   setLanguage('en');  // Switch to English
   setLanguage('es');  // Switch to Spanish
   setLanguage('fr');  // Switch to French
   setLanguage('pt');  // Switch to Portuguese
   setLanguage('de');  // Switch to German
   ```

5. **Current Language Check:**
   ```javascript
   let currentLang = getCurrentLanguage(); // Returns current language code
   ```

## Adding New Pages

When you create new HTML pages, simply:
1. Include the same language selector HTML and styles after `<body>` opening tag
2. Include `<script src="translations.js" defer></script>` in the head
3. Use `data-t` attributes on elements you want to translate

## Support for Dynamic Content

If you add new text to your pages dynamically with JavaScript:
```javascript
// After creating new element
const newElement = document.createElement('span');
newElement.setAttribute('data-t', 'dashboard');
newElement.textContent = 'Dashboard';

// Trigger translation update
updatePageTranslations();
```

## Troubleshooting

**Selector not appearing?**
- Check browser console for JavaScript errors
- Verify translations.js is loading
- Check that language-select element ID is correct

**Language not changing?**
- Check browser localStorage is enabled
- Verify page is using latest translations.js
- Check browser console for errors

**Settings not persisting?**
- Browser's localStorage might be disabled
- Try incognito/private mode
- Clear cache and cookies if having issues
